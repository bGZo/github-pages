一开始看着请求标题全是 Untitled，但是原始 URL、Params、Body、Headers 都在，所以没有放在心上，能用就行。但是关闭请求的快捷键 ctrl+w 会把整个软件都关掉就很烦，于是决定升级APP。安装完毕之后，弹了一个红色的提示，说什么需要恢复，感觉事情不对，检查 Collections 内部的URL 全都变成 https://echo.hoppscotch.io 了。

关键我没有任何备份，然后我攒了两年的 API 全都没了。没有任何渠道可以恢复一年之前的URL。账号完全没有任何用，我真是草了。另一个逆天的设计是没有回收站，如果你只是误删除了 Collection，你没有任何渠道可以恢复，你能做到的就是一用一备份，然后祈祷你的右手或者猫不会不小心点到那个删除的按钮。

真的给我整破防了。

如果想要在手机上追求一点好的拍照效果的话，那么手机就难免遇到散热噩梦。

我们假定这是一款中高端手机，那么为了塞下更大的电池，主板就必然会设计成双层甚至是多层的，并且一定会放在发热镜头的下面。

不幸的是，这是一款中高端手机，CPU发热是难免的，所以我们一定要考虑导热，那么问题来了，机器后面被越来越大的相机模组霸占，那么导热的工作就自然前移，成为了前端屏幕的工作，这个时候屏幕除了内容展示外，甚至还需要充当均热板的作用。

这样的设计合理，但除了手指感知温度上升而带来的不适之外，如果你还需要加个物理散热的时候，这个方案就会变得非常滑稽。

因为相机已经把热源霸占了，你只能放在热源的下方，给电池散热。

如果你恰巧幸运，买的是一加，正好李杰还把磁吸砍了，还需要再外挂一个散热引磁壳才能派上用场，整个方案就会变得非常蠢。

当然最好降温的方法永远是在热源上做散热，就像是我现在正在把散热器放在屏幕上方打这段文字一样。

滑稽的设计，滑稽的我。
